## 1. 快速开始

### 1.1 注册账号

1. 打开浏览器访问系统地址（开发环境：`http://localhost:3000`）
2. 点击登录页底部的 **"Create one"** 跳转到注册页
3. 填写以下信息：
   - **Username** — 登录用户名（3-50 个字符，不可重复）
   - **Email** — 邮箱地址（不可重复）
   - **Password** — 密码（至少 8 位，需包含大写字母、小写字母、数字）
4. 点击 **"Create Account"** 提交
5. 注册成功后自动跳转登录页

### 1.2 登录

1. 输入用户名和密码
2. 查看验证码图片（数学题，如 `13 + 7 = ?`），在输入框中填入答案
3. 点击 **"Sign In"** 登录
4. 登录成功后进入主界面

> **提示**：点击验证码图片可以换一张新的。验证码 5 分钟内有效。

### 1.3 界面概览

登录后进入主界面（Dashboard），分为三个区域：

```
┌──────────────────────────────────────────────┐
│  ProjectFlow  │  搜索...  │  🔔  👤 demo     │  ← 顶部导航栏
├────────────┬─────────────────────────────────┤
│ 项目列表   │  Todo │ In Progress │ Review │ … │  ← 看板
│ ERP系统    │ ┌────┐│             │        │   │
│ 协作平台   │ │卡片││             │        │   │
│ AI知识库   │ └────┘│             │        │   │
│            │       │             │        │   │
│ [+New]     │       │             │        │   │
└────────────┴─────────────────────────────────┘
```

- **顶部导航栏**：Logo、搜索框、通知铃铛（红色数字 = 未读数）、用户头像
- **左侧边栏**：项目列表，点击切换项目
- **主区域**：Kanban 看板（5 列），可拖拽任务卡片

## 2. 用户角色说明

| 角色 | 权限 |
|---|---|
| **Guest**（游客） | 只能看登录页和注册页，未登录无任何操作权限 |
| **Developer**（开发者） | 查看项目、管理任务、上传附件、发表评论 |
| **ProjectManager**（项目经理） | 全部权限：创建/审批/归档项目、管理成员、删除任务、查看日志 |

> **注意**：新注册用户默认为 **Developer**。如需项目经理权限，请联系系统管理员升级。

## 3. 项目管理

### 3.1 创建项目

> 需要 **ProjectManager** 角色

1. 点击左侧边栏或项目列表页的 **"+ New Project"** 按钮
2. 输入 **Project Name**（必填）和 Description（可选）
3. 点击提交

创建后项目状态为 **Draft（草稿）**。

### 3.2 项目状态流转

```
Draft ──提交──→ PendingApproval ──审批──→ Approved ──开始──→ Developing ──完成──→ Completed ──归档──→ Archived
```

| 状态 | 含义 | 谁可以操作 |
|---|---|---|
| Draft | 草稿 | 创建者编辑、提交审批 |
| PendingApproval | 待审批 | PM 审批通过 |
| Approved | 已批准 | 任意成员开始开发 |
| Developing | 开发中 | 任意成员完成任务 |
| Completed | 已完成 | PM 归档 |
| Archived | 归档 | 只读 |

操作按钮在项目详情页（`/projects/:id`）的顶部卡片中：
- **Submit for Approval** — 提交审批
- **Approve** — 审批通过（PM 可见）
- **Start Development** — 开始开发
- **Mark Complete** — 标记完成
- **Archive** — 归档（PM 可见）

### 3.3 添加项目成员

1. 进入项目详情页（侧边栏点击项目名）
2. 找到 **Team Members** 区域
3. 点击 **"+ Add"** 按钮（PM 可见）
4. 输入对方 **User ID**（数字），点击 Add

> **如何找到 User ID？** 用户 ID 在注册时系统自动分配。目前可在数据库 `users` 表中查看。

### 3.4 编辑项目

在项目详情页点击 **"Edit"** 按钮（PM 可见），修改名称或描述后点击 Save。

## 4. 任务管理

### 4.1 创建任务

1. 在看板页或项目详情页
2. 使用 API 或前端功能创建任务
3. 填写：
   - **Title** — 任务标题（必填）
   - **Description** — 任务描述
   - **Priority** — 优先级：Low / Medium / High / Critical
   - **Deadline** — 截止日期（可选）
   - **Assignee** — 指派给（可选）

### 4.2 任务优先级

| 优先级 | 颜色 | 含义 |
|---|---|---|
| Low | 绿色 | 低优先级，可延后 |
| Medium | 橙色 | 普通优先级 |
| High | 红色 | 高优先级，尽快处理 |
| Critical | 深红加粗 | 紧急，优先处理 |

### 4.3 任务状态流转

```
Todo ──→ InProgress ──→ Review ──→ Testing ──→ Done
```

| 操作 | 说明 |
|---|---|
| Todo → InProgress | 开始开发 |
| InProgress → Review | 提交代码审查 |
| Review → Testing | 进入测试 |
| Testing → Done | 测试通过 |

> **关键规则**：状态**必须按顺序流转**，不能跳步。从 Todo 不能直接到 Review，必须先经过 InProgress。

> **乐观锁保护**：多人同时编辑同一任务时，先提交的生效，后提交的会收到提示"数据已被修改，请刷新重试"，防止相互覆盖。

### 4.4 操作任务

点击看板上的任务卡片，弹出详情窗口，可以：
- 查看任务详情和描述
- 点击 **"Move to [下一状态]"** 流转状态
- 编辑标题、描述、优先级、截止日期
- 指派给其他成员
- 查看和发表评论
- 上传附件

## 5. 看板使用

### 5.1 看板布局

看板分为 5 列，对应任务状态：

| 列 | 对应状态 |
|---|---|
| To Do | Todo |
| In Progress | InProgress |
| Review | Review |
| Testing | Testing |
| Done | Done |

### 5.2 筛选功能

看板顶部提供两个筛选器：
- **Priority** — 按优先级筛选（All / Low / Medium / High / Critical）
- **Assignee** — 按负责人筛选

### 5.3 自动刷新

看板每 **5 秒** 自动刷新数据，无需手动刷新页面。

## 6. 评论协同

### 6.1 发表评论

- **项目评论**：在项目详情页的 Comments 区域直接输入
- **任务评论**：点击任务卡片，在弹窗底部的评论区输入

### 6.2 回复评论

点击评论下方的回复按钮，输入内容即可嵌套回复。

### 6.3 @ 提醒

在评论中输入 `@username` 可以提醒特定用户（例如：`@zhangsan 请看一下这个任务`），被 @ 的用户会收到站内通知。

### 6.4 删除评论

只有评论的**原作者**可以删除自己的评论。

## 7. 文件附件

### 7.1 上传文件

1. 打开任务详情弹窗
2. 在附件区域选择文件上传

### 7.2 支持的文件类型

| 类型 | 扩展名 |
|---|---|
| 图片 | jpg, png, gif, webp 等 |
| 文档 | pdf, doc, docx |
| 压缩包 | zip, rar |
| 文本 | txt |

### 7.3 限制

- 单文件最大 **20MB**
- 文件名自动添加 UUID 前缀，避免同名冲突

## 8. 通知系统

### 8.1 通知触发场景

| 场景 | 通知内容 |
|---|---|
| 项目审批通过 | "您的项目已通过审批" |
| 任务被分配 | "您被分配了任务 XXX" |
| 评论被回复 | "XXX 回复了您的评论" |
| @ 提醒 | "XXX 在评论中提到了您" |
| 状态变更 | "任务 XXX 状态变更" |

### 8.2 查看通知

1. 点击顶部导航栏的 **铃铛图标** 🔔
2. 红色数字表示未读通知数
3. 点击通知可标记为已读
4. 点击 **"Mark all read"** 全部标记已读

## 9. 常见问题

### Q1：我注册了但看不到任何项目？

新注册用户默认是 **Developer** 角色。你需要被项目经理添加为项目成员后才能在侧边栏看到项目。请联系 PM 将你添加到相关项目中。

### Q2：为什么我点 Review 报错 "invalid status transition"？

任务状态必须**按顺序流转**。从 Todo 只能到 InProgress，不能直接跳到 Review。请依次点击 "Move to InProgress" → "Move to Review"。

### Q3：为什么我修改任务后提示"数据已被修改"？

这是**乐观锁保护**。你打开任务期间，另一个用户已经修改了同一条任务。请刷新页面查看最新状态后重新操作。

### Q4：验证码看不清怎么办？

点击验证码图片即可换一张新的。每题都是随机生成的加减法，例如 `8 + 14 = ?` 或 `17 - 3 = ?`。

### Q5：我想创建项目但没有权限怎么办？

只有 **ProjectManager** 角色可以创建项目。请联系系统管理员将你的账号升级为 PM。

### Q6：搜索功能怎么用？

顶部搜索框输入关键词可以搜索项目或任务（当前版本为前端展示，后端搜索接口待实现）。
