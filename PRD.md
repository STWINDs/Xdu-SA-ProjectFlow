# ProjectFlow —— 软件研发协作平台

## 1. 项目背景

在软件研发过程中，项目经理需要对项目进行规划、审批和进度管理，开发者需要完成任务开发、提交成果并进行协同沟通。

传统通过微信群、Excel、邮件管理项目的方式存在以下问题：

* 项目状态不透明
* 任务分配混乱
* 进度跟踪困难
* 文件版本容易丢失
* 多人协同修改时容易产生冲突

因此设计一套轻量级研发协作平台，实现项目全生命周期管理。

# 2. 用户角色

## 游客（Guest）

未登录用户。

权限：

* 访问登录页面
* 注册账号
* 查看系统介绍

限制：

* 不可查看项目
* 不可查看任务
* 不可上传附件
* 不可评论

---

## 开发者（Developer）

参与项目开发。

权限：

* 查看参与项目
* 查看任务
* 修改任务状态
* 提交开发成果
* 上传附件
* 评论任务
* 查看项目进度

限制：

* 无法审批项目
* 无法删除项目
* 无法管理成员

---

## 项目经理（Project Manager）

项目负责人。

权限：

* 创建项目
* 审批项目
* 分配任务
* 删除任务
* 管理项目成员
* 查看统计报表
* 查看操作日志

# 4. 并发协同方案

采用乐观锁机制。

每条项目与任务记录维护：

version

示例：

Task A

version=10

用户A读取：

version=10

用户B读取：

version=10

用户A更新成功：

version=11

用户B提交：

version=10

更新失败

提示：

"当前数据已被其他用户修改，请刷新后重试"

避免最后写入覆盖问题。
